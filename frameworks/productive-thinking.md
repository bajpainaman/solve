# Productive Thinking Model (Tim Hurson)

**Phase:** Define · **Source:** https://untools.co/productive-thinking-model

## Entry Predicate
`always_run` — provides a 6-step backbone the other Define frameworks plug into.

## Inputs
- `intake.problem_refined`
- `intake.success_criteria`

## Method
1. **What's going on?** — current state, pain, who feels it.
2. **What's success?** — concrete description of solved state.
3. **What's the question?** — the right question to answer; often not the one originally asked.
4. **Generate answers** — divergent: as many candidate answers as possible (consume Zwicky archetypes).
5. **Forge the solution** — convergent: pick + refine the strongest 2-3.
6. **Align resources** — what people, time, capital, decisions are needed to ship.

## Output Schema
```
1. WHAT'S GOING ON
   <prose>
2. WHAT'S SUCCESS
   <prose with metrics>
3. THE QUESTION
   <single-sentence canonical question>
4. CANDIDATE ANSWERS (divergent, ≥ 7)
   - <answer>
   - <answer>
   ...
5. FORGED SOLUTIONS (convergent, top 2-3)
   - <solution> — strengths / risks
   - <solution> — strengths / risks
6. RESOURCE PLAN
   - people: <list>
   - time: <estimate>
   - decisions needed: <list>
```

## Decision Hook
Forged solutions feed Decision Matrix as options. Resource plan feeds Eisenhower (urgency) and Impact-Effort.

## What This Means For The Decision
Productive Thinking is a checklist that catches missing pieces. If step 6 (resources) is empty, the decision is premature.
