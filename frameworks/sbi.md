# Situation-Behavior-Impact (SBI)

**Phase:** Decide · **Source:** https://untools.co/situation-behavior-impact

## Entry Predicate
`intake.problem_refined contains interpersonal-conflict signals` (e.g. "team", "feedback", "manager", "report", "communication") OR `frameworks/conflict-resolution-diagram.md` was triggered.

## Method
For each interpersonal communication that the recommendation requires:
1. **Situation** — when/where the behavior occurred (concrete, not generalized).
2. **Behavior** — what was observed (objective, no judgment).
3. **Impact** — how it affected work, team, the speaker.

This gives the team a **script** for the conversations the decision implies.

## Output Schema
```
CONVERSATION 1: <who with, about what>
  S: In <specific meeting/event/time>...
  B: When you/we <observable action>...
  I: The impact was <specific consequence>...

CONVERSATION 2: ...
```

## Decision Hook
SBI scripts become part of the **execution plan** in the recommendation. If the recommendation requires hard conversations, the report includes the SBI scripts so the conversations actually happen.

## What This Means For The Decision
Decisions that require behavior change require explicit conversations. Without SBI scripts, "we'll talk to them about it" becomes a never-happens. With scripts, the conversation has a starting line.
