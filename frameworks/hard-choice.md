# Hard Choice Model (Ruth Chang)

**Phase:** Decide · **Source:** https://untools.co/hard-choice-model

## Entry Predicate
`intake.reversibility = one-way-door` OR `top 2 options in Decision Matrix within 5%` — only run on genuinely hard choices.

## Inputs
- Top 2 options from Decision Matrix

## Method
Hard choices are not better/worse — they're **on a par**. Standard cost-benefit fails. Apply Chang's framework:
1. Acknowledge the options are on a par (neither dominates on all dimensions).
2. Identify the **values** at stake on each side.
3. Ask: which option do I have **reasons to commit to as my own**?
4. The choice is **self-defining** — it's about who you (or the team) become by choosing.

## Output Schema
```
THE TWO OPTIONS
  A: <option>
  B: <option>

WHY THIS IS HARD (not better/worse)
  - A wins on: <values>
  - B wins on: <values>

WHO DOES EACH OPTION MAKE US?
  Choosing A: a team that <identity>
  Choosing B: a team that <identity>

COMMITMENT TEST
  - For A: can we commit our reasons + agency to A? <yes/no>
  - For B: can we commit our reasons + agency to B? <yes/no>

RECOMMENDATION
  <choice> — because we commit to becoming <identity>
```

## Decision Hook
The recommendation here **overrides** Decision Matrix when this framework is triggered. The Hard Choice answer is the recommendation; matrix scores become supporting evidence rather than the decision itself.

## What This Means For The Decision
Some decisions can't be optimized — they have to be committed to. Trying to find the "right" answer when one doesn't exist wastes time and erodes trust. Pick what you're willing to be defined by.
